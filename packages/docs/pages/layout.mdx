# Layout, Container and Grid

Any well-designed project should be done in a grid. This chapter describes how to code grid in your project.

## `Container`

Main element is `Container`. It doesn't do much, just creates a container with margins.

```javascript live=true fullscreen=true
<Container>
  <Box sx={{ bg: "mono300", p: 3 }}>Lorem ipsum dolor sit amet</Box>
</Container>
```

By default `Container` has `10vw` margin.

You can change default in a theme by setting `containers.main` value:

```javascript
// theme.js

export default {
  containers: {
    main: ["10px", null, "10vw"],
    secondary: ["30vw", null, null, "20vw"]
  }
  // remaining config...
};
```

As you see, apart from `main` we also defined `secondary` container. We can use it this way:

```javascript live=true fullscreen=true
<Container variant={"secondary"}>
  <Box sx={{ bg: "mono300", p: 3 }}>Lorem ipsum dolor sit amet</Box>
</Container>
```

If you want to contextually create a different `Container` and not put it into a theme, this is how you do it:

```javascript live=true fullscreen=true
<Container variant={["20px", "40px", "60px", "80px", "100px"]}>
  <Box sx={{ bg: "mono300", p: 3 }}>Lorem ipsum dolor sit amet</Box>
</Container>
```

You can easily add styles like background or margin to a `Container`:

```javascript live=true fullscreen=true
<Container sx={{ bg: "coral", py: 8 }}>
  <Box>Lorem ipsum dolor sit amet</Box>
</Container>
```
